第一次考试分析

准备开始补前面的知识了，先把前面几次考试题目分析一下

  1. 5层网络模型以及每层协议有些？

    应用层：HTTP、Telnet、SMTP、FTP、RIP、NFS、DNS

    传输层：TCP、UDP

    网络层：IP、ICMP、ARP、RARP、AKP、UUCP

    数据链路层：PPP、FDDI、Ethernet、Arpanet、PDN、SLIP

    物理层：IEEE 802.1A IEEE 802.2

  2. DNS工作过程
    DNS，全名为Domain Name System，域名系统，它是一个分布式数据库，存储域名和IP的一一对应关系，方便用户访问网络。能够将域名解析为IP（域名解析），也能将IP解析为域名（反向解析），该协议是基于UDP协议，默认端口是53。

    域名空间结构：




    如下图所示，展示了域名系统最为常见的结构，“.”是完全合格域名（FQDN）中最后的符号，分析域名系统是从后往左分析，“ftp.ichunqiu.com. ”，此处“.”可以当做根域，“com”是顶级域名，“ichunqiu”是二级域，“ftp”是主机名。




    在DNS工作过程，需要按照正向查询或者反向查询的方式进行，在正向查询过程中有递归查询和迭代查询两种方式，用户PC向本地DNS服务器的查询是递归，本地DNS服务器向其他的域名服务器（根DNS、.COM DNS等）的查询是迭代，具体的查询示意图如下。



  3. 端口

    http端口：80

    DNS端口：53

    SSH端口：22

    FTP端口：20,21

    mysql端口：3306

    Redis数据库端口：6379

    Mssql数据库端口：1433

  4. 操作系统有哪些
    两大类：类Unix和Windows

    类Unix系统：

    • Redhat，基于Redhat出现了Centos。
    • Debian，基于Debian出现了Ubuntu、kali、Deepin

    Windows系统：Win XP、Win7、Win10、Windows Server

    Windows Server常被用作服务器系统，比较老的有2003、2008、2012，常被用来搭建IIS,SQLServer服务器。

  5. 虚拟机的三种网络连接方式是什么？其区别？

    • NAT模式：VM会启用VM DHCP服务，会给我们选择NAT模式的机器分配IP。客户端PC会自动获取IP，可以连接到外网，可以ping通宿主机（宿主机上会多出来两个网卡：NAT和仅主机，NAT模式网卡上的IP和DNCP中设置的地址池是同一网段）
    • 桥接模式：桥接模式的PC和宿主机是平等，会自动获取宿主机所在网络的IP，能ping通宿主机和外网
    • 仅主机模式：虚拟机选择仅主机模式后会自动获取IP，该IP在虚拟网络编辑器中可以设置，无法连接外网，选择仅主机模式的虚拟机之间能够相互ping通，这样的虚拟机只能ping通宿主机的VMnet1这个网卡

  6. Windows命令：

    查看系统任务列表：tasklist

    获取系统详细信息：systeminfo

    创建文件并往文件中写数据：echo 123 > test.txt

    读文件：type test.txt

    创建用户test，并添加管理组：net user test 123 /add net localgroup test /add

  7. Linux命令：

    系统重启命令：reboot

    显示所有文件的详细信息：ls -al

    创建软链接：ln -s 源文件 目标文件

    查看当前工作目录：pwd

    切换到root家目录：cd ~

    查看IP配置信息：ifconfig

    centos的网卡配置文件在哪里：跟目录下/etc/sysconfig/network-scripts/

    读物文件ichunqiu.txt 前20行文件：head -n 20 ichunqiu.txt

    将文件pen.koo改名字为pen.ook：mv pen.koo pen.ook

    在linux的/var/www/下找到名字以hacker开头的文件：find /var/www/ hacker

    在centos7中关闭防火墙的三种方法：service firewalld stop

    vim编辑器的三种模式：

    • 命令模式：打开一个文件就是命令，输入：q 是退出 输入 ：wq 是保存退出 :w 保存

    • 输入模式：可以写入数据的模式，这种模式的进入方法：i(在光标的位置输入) a(光标后一位输入) o(另起一行输入) I A O,退出来摁 ESC 键
    • 底部命令模式：通过底部命令去控制文档。需要在命令模式下输入：即可进入该模式
      set nu 设置行号
set nonu 取消行号<br>

    在VIM中将当前行中所有的key1替换成key2：

    ：%s/key1/key2 所有的文档每行第一次出现的key1替换成key2

    ：%s/key1/key2/g 所有的key1替换成key2

    权限710是什么意思：

    当前文件所有者有可读可写可操作权限

    当前文件所属组有可执行权限

    其他人对该文件的权限是不可读不可写不可执行

    将权限为651修改为511：

    当前文件所有者权限从可读可写变成可读可执行

    当前文件所属者权限从可读可执行变成可执行

    其他人对该文件的权限没变，都只有可执行权限。

    添加用户user指定用户ID为200、组为demo_group、解析器为/bin/bash：useradd -u 200 -g demo_group -s /bin/bash user

    源码安装的过程：

    step 1:下载redis的源码，然后解压缩 tar -zxvf redis.tar.gz

    step 2:cd 切换进入该文件夹，

    step 3:./configure –prefix=/usr/local/redis[可选]

    step 4 :make 编译

    step 5 :make install 安装软件
